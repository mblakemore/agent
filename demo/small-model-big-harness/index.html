03 · what we tried

Every lever measured — kept the winners, dropped the rest

The rule underneath: disk, cores and memory are all saturated at the optimum → only remove waste, add bandwidth, or work during an idle phase helps. Every failed lever broke it.

04 · route by comparative advantage

Fast model drives every turn · deep model consulted only when stuck

Three stall detectors auto-call the deep advisor and inject its answer. Deep advice is expensive (≈2 min/call) → it fires only when the driver is genuinely stuck.

05 · trust replaced by verification

The model can code — it just can't be trusted to say "done." So the harness checks.

✓ gate
rejects "verified / done / committed"
the trace doesn't support
19/19
catastrophic commands blocked
(rm -rf /, raw-disk, fork bombs)
18/18
safe commands still allowed
(rm -rf ./build works)

Capability was never the gap (Phase B–C above). The risk is the ~1 in 15 time the model claims "verified" having run nothing — rare, but invisible from the output. So the harness runs the success check itself and, on an unsupported claim, injects "there is no such tool call — run it now and show the output." Trust is replaced by verification — model-independent, local or frontier.

06 · the honest negative

Built the whole advisor tier — measured that it doesn't lift this model yet

The tier is fully operational and shows no positive lift on this strong 27B — stated, not buried. When it was called the advice was real (it once spotted an import warning hiding the true traceback, and independently reproduced a fix a human had just made) — it simply didn't move the aggregate on a model already this disciplined. Building it well enough to prove that beats shipping a feature that quietly hurts.

07 · what more hardware buys — measured, not guessed

Storage and memory are the levers · not a bigger GPU

The GPU levers were both measured and dropped. The upgrade this workload responds to is disk width and RAM — and we know because we measured it, not because we assumed.

the one idea

Scaffold the harness, don't retrain the model

Route hard problems to a deeper model · run the tests yourself instead of trusting "done" · guard the destructive commands · measure honestly enough to find the feature that doesn't help. The harness is where the leverage lives — for a local model and a frontier model alike.
